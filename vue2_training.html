1. Vue2简介

1.1 什么是Vue2？

Vue2是一个渐进式JavaScript框架，用于构建用户界面。它的核心库只关注视图层，易于上手，同时也便于与第三方库或已有项目整合。

Vue2的核心特点：

  • 声明式渲染：用简洁的模板语法来声明式地将数据渲染进DOM
  • 组件化：将界面抽象为一个个可复用的组件
  • 响应式：数据变化时，视图会自动更新
  • 渐进式：可以逐步采用，不需要重写整个应用

1.2 Vue2的优势

// 传统JavaScript操作DOM的方式
document.getElementById('app').innerHTML = '你好，' + name;

// Vue2的声明式方式
<template>
  <div>你好，{{ name }}</div>
</template>

Vue2让我们从繁琐的DOM操作中解脱出来，专注于数据和逻辑。

3. Vue实例与生命周期

3.1 创建Vue实例

const vm = new Vue({
  // 选项对象
  el: '#app',           // 挂载点
  data: {               // 数据
    message: 'Hello Vue!'
  },
  methods: {            // 方法
    greet() {
      alert(this.message)
    }
  }
})

重要概念解释：

  • el：Vue实例挂载的DOM元素
  • data：Vue实例的数据对象，这些数据是响应式的
  • methods：定义方法，可以在模板中调用

3.2 生命周期详解

Vue实例从创建到销毁会经历一系列的生命周期钩子：

new Vue({
  el: '#app',
  data: {
    message: 'Hello'
  },
  
  // 实例初始化之后，数据观测之前调用
  beforeCreate() {
    console.log('beforeCreate: 实例刚创建，data还未初始化')
  },
  
  // 实例已经创建完成，data已初始化
  created() {
    console.log('created: 数据初始化完成，可以访问data')
    // 适合：发送网络请求、初始化数据
  },
  
  // 挂载开始之前调用
  beforeMount() {
    console.log('beforeMount: 模板编译完成，即将挂载到DOM')
  },
  
  // 实例挂载完成
  mounted() {
    console.log('mounted: DOM挂载完成，可以访问DOM元素')
    // 适合：操作DOM、启动定时器、绑定自定义事件
  },
  
  // 数据更新前调用
  beforeUpdate() {
    console.log('beforeUpdate: 数据更新前')
  },
  
  // 数据更新后调用
  updated() {
    console.log('updated: 数据更新后，DOM已重新渲染')
  },
  
  // 实例销毁前调用
  beforeDestroy() {
    console.log('beforeDestroy: 实例销毁前')
    // 适合：清理定时器、解绑事件、清理订阅
  },
  
  // 实例销毁后调用
  destroyed() {
    console.log('destroyed: 实例已销毁')
  }
})

生命周期应用场景：

  • created：发送Ajax请求获取数据
  • mounted：操作DOM、初始化第三方插件
  • beforeDestroy：清理工作，防止内存泄漏

4. 模板语法

4.1 插值表达式

<!-- 文本插值 -->
<div>{{ message }}</div>

<!-- 表达式 -->
<div>{{ number + 1 }}</div>
<div>{{ ok ? 'YES' : 'NO' }}</div>
<div>{{ message.split('').reverse().join('') }}</div>

<!-- 调用方法 -->
<div>{{ formatDate(date) }}</div>

注意事项：

  • 插值表达式只能包含单个表达式
  • 不能使用语句（如if语句）
  • 不能访问全局变量（如window）

4.2 原始HTML

<template>
  <div>
    <!-- 会被转义，显示HTML标签 -->
    <p>{{ rawHtml }}</p>
    
    <!-- 不会被转义，渲染为HTML -->
    <p v-html="rawHtml"></p>
  </div>
</template>

<script>
export default {
  data() {
    return {
      rawHtml: '<span style="color: red">这是红色文字</span>'
    }
  }
}
</script>

**安全提醒：**永远不要对用户提供的内容使用v-html，这可能导致XSS攻击。

4.3 属性绑定

<!-- 绑定属性 -->
<div v-bind:id="dynamicId"></div>
<div v-bind:class="className"></div>

<!-- 简写形式 -->
<div :id="dynamicId"></div>
<div :class="className"></div>

<!-- 动态属性名 -->
<div :[attributeName]="value"></div>

5. 指令详解

5.1 v-if / v-else / v-else-if

<template>
  <div>
    <div v-if="type === 'A'">A类型</div>
    <div v-else-if="type === 'B'">B类型</div>
    <div v-else>其他类型</div>
    
    <!-- template标签不会渲染到DOM中 -->
    <template v-if="loginType === 'username'">
      <label>用户名</label>
      <input placeholder="输入用户名" key="username-input">
    </template>
    <template v-else>
      <label>邮箱</label>
      <input placeholder="输入邮箱" key="email-input">
    </template>
  </div>
</template>

v-if vs v-show的区别：

  • v-if：条件性渲染，条件为false时元素不存在于DOM中
  • v-show：始终渲染，只是通过CSS的display控制显示隐藏

5.2 v-for

<template>
  <div>
    <!-- 遍历数组 -->
    <li v-for="(item, index) in items" :key="item.id">
      {{ index }} - {{ item.name }}
    </li>
    
    <!-- 遍历对象 -->
    <li v-for="(value, key, index) in object" :key="key">
      {{ index }}. {{ key }}: {{ value }}
    </li>
    
    <!-- 遍历数字 -->
    <span v-for="n in 10" :key="n">{{ n }}</span>
  </div>
</template>

<script>
export default {
  data() {
    return {
      items: [
        { id: 1, name: '苹果' },
        { id: 2, name: '香蕉' }
      ],
      object: {
        title: 'How to do lists in Vue',
        author: 'Jane Doe',
        publishedAt: '2016-04-10'
      }
    }
  }
}
</script>

**重要提醒：**始终为v-for提供key属性，且key应该是唯一的标识符。

5.3 v-model

<template>
  <div>
    <!-- 输入框 -->
    <input v-model="message" placeholder="请输入">
    <p>{{ message }}</p>
    
    <!-- 复选框 -->
    <input type="checkbox" v-model="checked">
    <label>{{ checked }}</label>
    
    <!-- 单选框 -->
    <input type="radio" value="A" v-model="picked">
    <input type="radio" value="B" v-model="picked">
    <span>选择：{{ picked }}</span>
    
    <!-- 选择框 -->
    <select v-model="selected">
      <option disabled value="">请选择</option>
      <option>A</option>
      <option>B</option>
    </select>
  </div>
</template>

v-model修饰符：

<!-- 在"change"时而非"input"时更新 -->
<input v-model.lazy="msg">

<!-- 自动将用户的输入值转为数值类型 -->
<input v-model.number="age" type="number">

<!-- 自动过滤用户输入的首尾空白字符 -->
<input v-model.trim="msg">

5.4 事件指令 v-on

<template>
  <div>
    <!-- 完整语法 -->
    <button v-on:click="doThis">点击我</button>
    
    <!-- 缩写 -->
    <button @click="doThis">点击我</button>
    
    <!-- 传递参数 -->
    <button @click="say('hi')">Say hi</button>
    
    <!-- 访问原始DOM事件 -->
    <button @click="warn('Form cannot be submitted yet.', $event)">
      Submit
    </button>
  </div>
</template>

<script>
export default {
  methods: {
    doThis() {
      console.log('按钮被点击了')
    },
    say(message) {
      alert(message)
    },
    warn(message, event) {
      if (event) {
        event.preventDefault()
      }
      alert(message)
    }
  }
}
</script>

事件修饰符：

<!-- 阻止单击事件继续传播 -->
<a @click.stop="doThis"></a>

<!-- 提交事件不再重载页面 -->
<form @submit.prevent="onSubmit"></form>

<!-- 修饰符可以串联 -->
<a @click.stop.prevent="doThat"></a>

<!-- 只有修饰符 -->
<form @submit.prevent></form>

<!-- 添加事件监听器时使用事件捕获模式 -->
<div @click.capture="doThis">...</div>

<!-- 只当在 event.target 是当前元素自身时触发处理函数 -->
<div @click.self="doThat">...</div>

<!-- 点击事件将只会触发一次 -->
<a @click.once="doThis"></a>

6. 数据绑定与响应式原理

6.1 响应式数据

export default {
  data() {
    return {
      // 这些数据都是响应式的
      message: 'Hello',
      user: {
        name: '张三',
        age: 25
      },
      items: ['苹果', '香蕉']
    }
  },
  created() {
    // 修改响应式数据会触发视图更新
    this.message = 'Hi'
    this.user.name = '李四'
    this.items.push('橙子')
  }
}

6.2 响应式原理简解

Vue2使用Object.defineProperty来实现响应式：

// Vue2响应式原理简化版
function defineReactive(obj, key, val) {
  Object.defineProperty(obj, key, {
    get() {
      console.log(`获取${key}: ${val}`)
      return val
    },
    set(newVal) {
      if (newVal !== val) {
        console.log(`设置${key}: ${newVal}`)
        val = newVal
        // 触发视图更新
        updateView()
      }
    }
  })
}

function updateView() {
  console.log('视图更新了')
}

6.3 响应式注意事项

export default {
  data() {
    return {
      user: {
        name: '张三'
      },
      items: ['苹果']
    }
  },
  methods: {
    addProperty() {
      // ❌ 直接添加属性不是响应式的
      this.user.age = 25
      
      // ✅ 使用Vue.set添加响应式属性
      this.$set(this.user, 'age', 25)
    },
    updateArray() {
      // ❌ 直接索引赋值不是响应式的
      this.items[0] = '香蕉'
      
      // ✅ 使用数组方法或Vue.set
      this.$set(this.items, 0, '香蕉')
      // 或者
      this.items.splice(0, 1, '香蕉')
    }
  }
}

8. 计算属性与侦听器

8.1 计算属性 computed

export default {
  data() {
    return {
      firstName: 'Foo',
      lastName: 'Bar',
      items: [
        { id: 1, name: '苹果', price: 10, quantity: 2 },
        { id: 2, name: '香蕉', price: 5, quantity: 3 }
      ]
    }
  },
  computed: {
    // 基础计算属性
    fullName() {
      return this.firstName + ' ' + this.lastName
    },
    
    // 计算属性会基于响应式依赖进行缓存
    totalPrice() {
      console.log('计算总价') // 只有相关数据变化时才会重新计算
      return this.items.reduce((total, item) => {
        return total + item.price * item.quantity
      }, 0)
    },
    
    // 可读写的计算属性
    fullNameWritable: {
      get() {
        return this.firstName + ' ' + this.lastName
      },
      set(newValue) {
        const names = newValue.split(' ')
        this.firstName = names[0]
        this.lastName = names[names.length - 1]
      }
    }
  }
}

计算属性 vs 方法的区别：

  • 计算属性基于响应式依赖进行缓存，只有依赖发生改变时才会重新求值
  • 方法每次调用都会执行函数

8.2 侦听器 watch

export default {
  data() {
    return {
      question: '',
      answer: 'I cannot give you an answer until you ask a question!',
      user: {
        name: '张三',
        profile: {
          age: 25
        }
      }
    }
  },
  watch: {
    // 侦听基础数据类型
    question(newQuestion, oldQuestion) {
      this.answer = 'Waiting for you to stop typing...'
      this.debouncedGetAnswer()
    },
    
    // 侦听对象属性变化（浅侦听）
    'user.name'(newName, oldName) {
      console.log(`用户名从 ${oldName} 改为 ${newName}`)
    },
    
    // 深度侦听
    user: {
      handler(newUser, oldUser) {
        console.log('用户对象发生了变化')
      },
      deep: true
    },
    
    // 立即执行
    question: {
      handler(val) {
        console.log('question changed:', val)
      },
      immediate: true
    }
  },
  created() {
    // 使用 lodash 的防抖函数
    this.debouncedGetAnswer = _.debounce(this.getAnswer, 500)
  },
  methods: {
    getAnswer() {
      if (this.question.indexOf('?') === -1) {
        this.answer = 'Questions usually contain a question mark. ;-)'
        return
      }
      this.answer = 'Thinking...'
      // 模拟API调用
      setTimeout(() => {
        this.answer = 'Yes, it is a good question!'
      }, 1000)
    }
  }
}

8.3 计算属性 vs 侦听器

export default {
  data() {
    return {
      firstName: 'Foo',
      lastName: 'Bar'
    }
  },
  
  // ✅ 推荐：使用计算属性
  computed: {
    fullName() {
      return this.firstName + ' ' + this.lastName
    }
  },
  
  // ❌ 不推荐：使用侦听器实现同样的功能
  watch: {
    firstName(val) {
      this.fullName = val + ' ' + this.lastName
    },
    lastName(val) {
      this.fullName = this.firstName + ' ' + val
    }
  }
}

选择建议：

  • 需要基于现有数据计算新数据时，使用计算属性
  • 需要在数据变化时执行异步操作或开销较大的操作时，使用侦听器

9. 条件渲染与列表渲染

9.1 条件渲染高级用法

<template>
  <div>
    <!-- v-if 在template上 -->
    <template v-if="loginType === 'username'">
      <label>Username</label>
      <input placeholder="Enter your username" key="username-input">
    </template>
    <template v-else>
      <label>Email</label>
      <input placeholder="Enter your email address" key="email-input">
    </template>
    
    <!-- 用key管理可复用的元素 -->
    <div v-if="showName" key="name">
      <label>姓名</label>
      <input>
    </div>
    <div v-else key="email">
      <label>邮箱</label>
      <input>
    </div>
    
    <!-- v-show -->
    <h1 v-show="ok">Hello!</h1>
  </div>
</template>

合并策略：

  • 数据对象：递归合并，组件数据优先
  • 生命周期钩子：合并为数组，都会被调用，混入的先调用

// 单元测试示例
import { shallowMount } from '@vue/test-utils'
import UserCard from '@/components/UserCard.vue'

describe('UserCard.vue', () => {
  const user = {
    id: 1,
    name: 'John Doe',
    email: 'john@example.com'
  }
  
  it('renders user name correctly', () => {
    const wrapper = shallowMount(UserCard, {
      propsData: { user }
    })
    
    expect(wrapper.find('.user-name').text()).toBe('John Doe')
  })
  
  it('emits click event when clicked', async () => {
    const wrapper = shallowMount(UserCard, {
      propsData: { user }
    })
    
    await wrapper.find('.user-card').trigger('click')
    
    expect(wrapper.emitted().click).toBeTruthy()
    expect(wrapper.emitted().click[0]).toEqual([user])
  })
})
